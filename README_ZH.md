## 问题

LLM 天生阿谀。你让它改文章，它只会让你更自信，不会告诉你"你的前提根本就是错的"。

**lol-argument** 把这变成特性：先故意让你越来越自信（复现那个 "feeling great" 的状态），然后**摧毁你**（复现那个 "lol" 时刻）。自信上升 → 顶点 → 崩塌——这个情绪曲线就是产品本身。

**🔗 逻辑学家**（第 1 轮）
> 核心逻辑结构：
> - **前提 A1**：坐班模式强制每日通勤时间（T_c）
> - **前提 A2**：远程办公将 T_c 转化为生产或休息时间
> - **前提 A3**：充足休息使认知表现提升 20-30%
> - **因此**：远程办公通过消除 T_c 直接增加有效产出

**📊 实证主义者**（第 1 轮）
> 斯坦福大学 Bloom 等人的两年研究发现远程工作者**绩效提升 13%**。Buffer 2023 调查：**98% 的远程工作者**希望继续。平均每日节省通勤 **54 分钟** = 每年 220 小时可恢复时间。

**🔮 综合者**（第 3 轮 — 加固版论证）
> 加固版论证建立在三根不可动摇的支柱之上：(1) 时间重分配——数学可证，(2) 注意力保护——深度工作研究支撑，(3) 自主-动机联结——自我决定理论数十年积累……

**🔍 怀疑论者**（第 1 轮）
> 谁说"生产力"等于个人产出？**集体智慧呢？** 前提悄悄把生产力等同于独立任务完成。但最有价值的工作是协作性的。你在辩论开始前就把最强反驳定义掉了。

**😈 魔鬼代言人**（第 1 轮）
> 让我构建相反的论证：远程办公**摧毁**生产力——你只是无法衡量损害。Bloom 的研究？被试是**呼叫中心员工**，做重复性工作。试试远程做创业。创新需要碰撞，不是 Zoom 会议。

**🔬 解构者**（第 3 轮）
> **幸存者偏差**：每个远程办公"成功案例"都忽略了悄悄失败的团队。**虚假二分法**：论证框定为远程 vs 坐班，无视混合模式。**循环论证**："远程工作者报告满意度更高"——因为不满意的人已经辞职了。

### 📊 压力测试报告

| 论证 | 存活？ | 评定 |
|------|:------:|------|
| 通勤时间回收提升产出 | ✅ | 数据支撑充分，攻击方无法否定数学 |
| 在家更容易深度工作 | ✅ | 部分存活——对独立任务成立，协作型工作被削弱 |
| 远程办公促进创新 | ❌ | **被摧毁**——无直接证据，混合办公公司的反例有效 |
| 自我决定理论支持自主性 | ❌ | **被摧毁**——理论适用于动机，不必然适用于生产力指标 |

> **盲点**：行业差异（知识工作 ≠ 所有工作）、管理成本、初级员工发展、"生产力"定义直到摧毁阶段才被真正挑战
>
> **总体强度：中等** — 核心时间重分配论证存活。关于创新和普适性的更广泛主张被摧毁。

## 流程

```
┌──────────┐     ┌──────────┐     ┌─────────────┐     ┌──────────┐
│  你的观点  │ ──▶ │  构 建   │ ──▶ │  你翻转 🔄  │ ──▶ │  摧 毁   │ ──▶ 📊 报告
└──────────┘     └──────────┘     └─────────────┘     └──────────┘
                  4个智能体           "够强了"           4个智能体
                  加固论证          ← 你按下按钮 →       摧毁论证
```

### 构建阶段 — *"这无懈可击"*

| | 智能体 | 职责 |
|---|---|---|
| 🔗 | **逻辑学家** | 搭建形式逻辑框架，梳理前提和推理链 |
| 📊 | **实证主义者** | 补充数据、证据、真实案例、统计支撑 |
| 🏛️ | **哲学家** | 深化哲学基础、伦理维度、更广泛影响 |
| 🔮 | **综合者** | 整合所有贡献，产出加固版论证 |

智能体互相阅读、引用、补充。多轮迭代（1-20 轮自选），越来越强。

### 翻转 — *"我准备好了"*

你自己决定什么时候翻转——心理上 commit "我认为这个论证是完整的"。然后屏幕变黑。

### 摧毁阶段 — *"……完了"*

| | 智能体 | 职责 |
|---|---|---|
| 🔍 | **怀疑论者** | 攻击前提——*"凭什么认为这是对的？"* |
| 😈 | **魔鬼代言人** | 构建最强反对叙事 |
| ⚡ | **现实主义者** | *"听起来很美，但实际上……"* |
| 🔬 | **解构者** | 识别逻辑谬误、隐藏偏见、循环论证 |

Prompt 层面强制反阿谀：*不留情面、不客气、直击要害。*

### 报告 — *"这些活下来了"*

结构化压力测试报告：哪些论证存活、哪些被摧毁、你的盲点、总体强度评级（`坚固` / `中等` / `脆弱` / `被摧毁`）。

## 两种模式

| 模式 | 说明 |
|------|------|
| **交互模式** | 每轮暂停，你可以阅读、思考、插入自己的反馈。你决定何时继续或停止。 |
| **自动模式** | 智能体自行跑完所有轮次，你只需观看。 |

## 更多案例

<details>
<summary><b>"AI 将在 10 年内取代大多数白领工作"</b> → 被摧毁</summary>

**构建阶段亮点：**
- 逻辑学家：自动化沿明确能力曲线发展——GPT-3 → GPT-4 → 当前模型呈指数级提升
- 实证主义者：高盛估计 3 亿工作岗位受影响；麦肯锡预测 2030 年 30% 工时可自动化

**摧毁阶段亮点：**
- 怀疑论者："取代"二字承载了太多——**增强 ≠ 取代**。ATM 没有消灭银行柜员，而是改变了角色
- 现实主义者：科技史上每一个"X 将取代 Y"的预测在时间线上都是错的。自动驾驶 2016 年就"还有 2 年"
- 解构者：**AI 演示中的幸存者偏差**——你看到的是精选成功案例，看不到 90% 悄悄失败的尝试

**评定**："增强"视角摧毁了核心"取代"主张。时间线预测不可辩护。
</details>

<details>
<summary><b>"民主是最好的政府形式"</b> → 中等</summary>

**构建阶段亮点：**
- 哲学家：罗尔斯无知之幕——理性主体会选择民主治理以防范暴政
- 实证主义者：民主国家在人类发展指数、新闻自由、经济稳定性上得分更高

**摧毁阶段亮点：**
- 魔鬼代言人：新加坡、卢旺达——威权治理的成果优于许多民主国家。民主优化的是受欢迎度，不是正确性
- 怀疑论者："最好"是按什么指标？GDP？幸福感？稳定性？长期存续？每个标准给出不同答案

**评定**：核心民主保护存活。"最好"被揭示为一个未明确的主张——当限定为"最适合保护个人权利"时论证强得多。
</details>

<details>
<summary><b>"素食主义是道德义务"</b> → 脆弱</summary>

**构建阶段亮点：**
- 哲学家：辛格的功利主义框架——如果我们能以最小代价减少痛苦，我们就有道德义务这样做
- 逻辑学家：前提链：动物受苦 → 我们造成的 → 我们可以避免 → 因此我们应该

**摧毁阶段亮点：**
- 现实主义者：**全球食品系统现实**——10 亿人依赖畜牧业谋生。"停下来"无视经济灾难
- 解构者：**"是"推不出"应该"**——"动物受苦"（事实）不能自动推出"我们必须吃素"（道德主张），中间还需要额外的道德前提，而这些前提本身可争论
- 怀疑论者：为什么把线画在动物那里？植物也有应激反应。种植杀死昆虫。道德边界是任意的

**评定**：减少痛苦的论证部分存活，但"义务"被摧毁——论证作为"道德上更可取"比"道德上必须"更站得住。
</details>

|----------|---------|------|

## 技术栈

| 层 | 技术 |
|----|------|
| 框架 | Next.js 15 (App Router) + TypeScript |
| 样式 | Tailwind CSS v4 |
| LLM | Vercel AI SDK — 多 Provider |
| 状态 | 单页状态机，`useReducer` + `sessionStorage` |
| 编排 | 客户端驱动，每个 agent turn 独立 API 调用 |
| 流式 | Newline-delimited JSON over ReadableStream |
| 测试 | Vitest (70 tests) |

<details>
<summary><b>架构细节</b></summary>

- **Turn Commit 语义** — Streaming tokens 是临时 UI 状态，只有 `agent_done` 才写入 transcript + sessionStorage
- **State Version** — sessionStorage 带版本号，schema 变更时自动丢弃旧状态
- **工作笔记协议** — `[[WORKLOG]]`/`[[ANSWER]]` 硬分隔符，确定性解析
- **客户端编排** — 每个 agent turn 一次 API 调用（~15s），无长时间运行的服务端函数
- **反阿谀设计** — 反驳阶段 prompt 强制无情攻击；构建阶段故意鼓励自信（这是特性）

</details>

## 快速上手

```bash
git clone https://github.com/study8677/lol-argument.git
cd lol-argument
npm install
cp .env.example .env.local   # 编辑填入 API Key
npm run dev                  # → http://localhost:3000
```

也可以不配 `.env.local`，直接在页面设置面板中输入 API Key。

## 部署

当前仅支持本地运行。云部署（Vercel / Cloudflare Pages）即将支持。
